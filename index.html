React Rendering

  • Rendering engine for the browser is single threaded
  • Nearly all actions happen synchronously
  • Previously, React would block the entire thread as it calculated the tree.
  • blocking the main thread could still cause some applications to not feel fluid

React 16 manages the main thread and rendering using native browser APIs by intermittently checking to see if there is other work that needs to be performed.

async rendering means is that the rendering work can be split into chunks and spread out over multiple frames

Async Rendering

  • Core algorithm was rewritten
  • Rendering work can be split into chunks
  • Spread out over multiple frames
  • React computes part of the tree
  • Pause rendering to check if the main thread
  • Any paints or updates need to be performed
  • When paints/ updates completed, React begins rendering again.
  • Accomplished by a data structure called a ‚Äúfiber‚Äù

Prioritize Updates

  • Prioritize updates by importance
  • High priority updates jumps to front of line
  • Processed important stuff first
  • Like: keyboard input gets high priority
  • API response will get low priority


Fiber Explanation

key input. This is high priority because the user needs that immediate feedback to feel fluid as opposed to a low priority update like an API response

Why Fiber

  • Breaking the UI updates into smaller units of work
  • Better overall user experience is achieved
  • Allow the main thread to execute other necessary tasks
  • Provides a smoother interface
  • Better perceived performance

Error Handling

Previously, errors inside components would corrupt React‚Äôs state and provide cryptic errors on subsequent renders.

React 16 includes error boundaries will not only provide much clearer error messaging, but also prevent the entire application from breaking.

After being added to your app, error boundaries catch errors and gracefully display a fallback UI without the entire component tree crashing. The boundaries can catch errors during rendering, in lifecycle methods, and in constructors of the whole tree below them. Error boundaries are simply implemented through the new lifecycle method componentDidCatch(error, info).
